# Banking Fraud Analytics Project Plan

## Objective

Develop an end-to-end banking fraud analytics solution using PostgreSQL and the IBM Synthetic Banking Dataset.

The project focuses on designing a relational database, validating and cleaning transactional data, performing exploratory and business-focused SQL analysis, investigating fraud patterns, and optimizing SQL queries.

---

## Dataset

The project uses three raw datasets:

- accounts.csv
- transactions.csv
- alerts.csv

---

## Technologies

- PostgreSQL
- pgAdmin
- SQL
- Git
- GitHub

---

## Project Workflow

1. Understand the data
2. Design the database
3. Create the PostgreSQL schema
4. Import raw data
5. Validate data
6. Clean data
7. Perform exploratory analysis
8. Perform fraud analytics
9. Optimize SQL queries
10. Document findings

---

## Expected Outcomes

- Relational PostgreSQL database
- Clean and validated data
- Business insights
- Fraud analysis
- Optimized SQL queries
- Professional GitHub portfolio
